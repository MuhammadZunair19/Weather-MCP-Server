# Security Policy

## Reporting a Vulnerability

Please do not open a public issue for suspected security vulnerabilities.

Instead, report the issue privately to the project maintainer through the repository hosting platform's private reporting channel if available.

Include:

- A description of the issue
- Steps to reproduce it
- Potential impact
- Any suggested mitigation

## Scope

This project is a demo-style MCP server and currently has:

- No authentication
- No persistent storage
- No secret management

Please still report vulnerabilities responsibly, especially around dependency, container, or unsafe input-handling concerns.
